# OSI Model Flow

Sending data: 7 → 1  
Receiving data: 1 → 7  
Direction depends on communication

## Layers:

7. Application → user/program (Chrome, apps)
6. Presentation → format + encryption (encode/decode)
5. Session → manages connection (start/keep/end session)
4. Transport → TCP / UDP (reliability, segmentation)
3. Network → IP + routing (find path)
2. Data Link → MAC address (local delivery)
1. Physical → cable + signal (bits)
